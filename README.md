# Hospital Management System (HMS) Prototype 🏥

Welcome to the **Hospital Management System (HMS)** repository! This project is a lightweight, frontend-driven desktop prototype designed to simulate hospital record management. 

Instead of jumping into complex external databases or cloud services right away, I focused purely on mastering application-level logic, handling dynamic data states, and executing complete **CRUD (Create, Read, Update, Delete)** workflows within the Windows Forms architecture.

## 🚀 Features
- **Dynamic CRUD Operations:** Add, view, update, and delete patient/hospital records seamlessly through the UI.
- **State Management:** Handles data modification dynamically inside the application runtime.
- **Interactive UI:** Simple, intuitive form layout designed for hospital desk operations.

## 🛠️ Tech Stack & Concepts Covered
- **Language:** C#
- **Framework:** .NET / Windows Forms (WinForms)
- **Concepts:** Event handling, data binding basics, UI state management, and algorithmic CRUD flow inside `Form1.cs`.

## 📂 Project Structure
To keep the foundation strong and minimal, the entire system is built utilizing:
- `Form1.cs` (CRUD logic, event handlers, and data handling)
- `Form1.designer.cs` (Form structure, inputs, buttons, and display grids)
